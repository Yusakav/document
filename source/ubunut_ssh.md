# Ubuntu SSH 免密登录配置指南

本指南详细介绍了如何通过 RSA 密钥认证方式实现从 **客户端 (Client)** 免密码远程登录到 **服务端 (Server)**，提高操作效率并增强安全性。

## 1. 准备工作：安装 SSH 服务
在开始之前，确保您的 **服务端** 已经安装并启动了 SSH 服务。

* **安装命令**：
    ```bash
    sudo apt-get update
    sudo apt-get install openssh-server
    ```
* **检查状态**：
    ```bash
    service ssh status
    ```

---

## 2. 客户端生成 RSA 密钥对
在您的 **本地机器（客户端）** 终端中执行以下步骤，生成身份验证所需的公钥和私钥。

* **执行命令**：
    ```bash
    ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
    ```
    * `-t rsa`：指定使用 RSA 加密算法。
    * `-P ''`：设置空密码（实现真正的自动化免密）。
    * `-f ~/.ssh/id_rsa`：指定密钥保存路径。

* **结果说明**：
    执行后，在 `~/.ssh/` 目录下会生成两个文件：
    1.  `id_rsa`：**私钥**，保存在本地，请勿泄露。
    2.  `id_rsa.pub`：**公钥**，需要发送给服务端。

---

## 3. 将公钥添加至服务端
将客户端生成的公钥内容，存入服务端目标用户的授权列表中。

### 步骤 A：服务端环境准备
在 **服务端** 对应用户（如 root）的家目录下执行：
```bash
mkdir -p ~/.ssh
touch ~/.ssh/authorized_keys
```

### 步骤 B：同步公钥内容
1.  **复制** 客户端 `~/.ssh/id_rsa.pub` 文件中的全部字符串。
2.  **粘贴** 到服务端 `~/.ssh/authorized_keys` 文件的末尾。

### 步骤 C：设置关键权限（非常重要）
为了系统安全，SSH 对文件权限有严格要求。如果权限过大，免密登录将失效。
```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

---

## 4. 重启服务与验证
完成配置后，需要重启服务端的 SSH 服务以使更改生效。

1.  **重启服务**（服务端）：
    ```bash
    service sshd restart
    ```

2.  **验证登录**（客户端）：
    在客户端尝试连接，将 `root` 和 `IP` 替换为实际信息：
    ```bash
    ssh root@192.168.1.133
    ```
    > **成功标志**：系统直接进入远程服务器命令行，不再提示输入密码。

---

## 5. 进阶技巧：多服务器批量配置
如果您需要同时管理多台服务器：
* **方法**：先将公钥添加至本地客户端的 `authorized_keys` 文件中。
* **快捷操作**：直接使用 `scp` 命令将该文件拷贝至各台服务器的对应目录，随后统一重启 `sshd` 服务即可。
